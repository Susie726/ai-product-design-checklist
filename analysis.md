# 从旅行助手类 AI 产品对比，提炼一份 AI 产品设计 Checklist

## 一、背景：为什么旅行 AI 特别适合做样本

旅行助手类 AI 是一个很好的研究样本，因为它同时具备：

- 决策复杂度高
- 约束条件多
- 信息时效性强
- 内容种草与交易转化并存
- 行前、行中、行后链路完整

也因此，旅行 AI 产品里出现的问题和优秀设计，几乎都可以迁移到更广义的 AI 助手、Agent、Copilot、垂类智能产品上。

在对携程、美团、马蜂窝、飞猪等产品的对比中，可以看到一个很关键的事实：

> AI 产品不是统一长成一个样子，而是被业务目标塑形。

如果业务目标是交易转化，产品会长成导购型 AI；如果业务目标是决策支持，产品会长成规划型 AI；如果业务目标是完成复杂任务，则需要更强的 Agent 架构、记忆能力和工具调度能力。

---

## 二、先分清楚：你做的是哪类 AI 产品？

这是所有设计之前最重要的问题。

### 1. 决策型 AI

这类产品的核心价值是：

- 帮用户想清楚怎么做
- 降低决策成本
- 承接模糊需求
- 帮用户逐步收敛条件

典型特征：

- 内容和推荐较强
- 更重视解释、种草、规划
- 转化链条较长

### 2. 交易型 AI

这类产品的核心价值是：

- 缩短下单路径
- 降低跳失率
- 提升点击与成交效率
- 把 AI 变成导购入口

典型特征：

- 商品卡片、优惠、订单能力更强
- 输出更偏结构化和操作化
- 对“最后一公里转化”极其敏感

### 3. 执行型 AI

这类产品的核心价值是：

- 不只是给建议，而是把任务执行下去
- 调用工具、查询数据、完成动作
- 帮用户真正推进结果

典型特征：

- 强工具调用
- 强状态管理
- 强错误处理和可回退能力

### 4. 陪伴型 AI

这类产品的核心价值是：

- 增强互动频率
- 提供情绪价值或持续陪伴
- 建立人格感和长期关系

## 三、AI 产品设计 Checklist

下面这份清单，适合在做产品方案、设计评审、竞品分析、MVP 规划时直接使用。

---

## 1. 战略定位

### Checklist

- [ ] 我们做的是决策型、交易型、执行型还是陪伴型 AI？
- [ ] 第一核心指标是什么：停留、转化、任务完成、复访还是 GMV？
- [ ] AI 在整个业务链路中扮演什么角色：获客、种草、转化、服务还是履约？
- [ ] 我们是否在试图同时做太多事情，导致主线不清？

### 判断标准

如果这些问题答不清，后续所有交互和功能设计都会摇摆。

---

## 2. 目标用户与任务定义

### Checklist

- [ ] 目标用户是谁？小白、熟练用户、专业用户、企业用户？
- [ ] 用户的核心任务是不是足够具体？
- [ ] 用户需求是高频、高价值，还是低频但高痛感？
- [ ] 任务是否可以被明确判断“完成了没有”？

### 常见错误

很多团队只写“帮助用户提高效率”，这没有设计价值。真正的任务定义应该是：

- 帮用户筛选酒店
- 帮用户比较方案
- 帮用户生成行程
- 帮用户下单
- 帮用户处理复杂售后

---

## 3. 场景拆分

### Checklist

- [ ] 产品是否按用户旅程拆分场景，而不是只做一个总入口？
- [ ] 是否区分行前/行中/行后，或发现/决策/执行/复盘等阶段？
- [ ] 有没有识别出高频、即时、刚需的子场景？
- [ ] 这些场景里，哪些适合做快捷入口而不是聊天？

### 设计启发

高频刚需功能往往应该被做成：

- 腰带卡片
- 快捷入口
- 场景按钮
- 模板任务

而不是全部藏在聊天框里。

---

## 4. 输入设计

### Checklist

- [ ] 用户能否低门槛地开始使用？
- [ ] 是否提供推荐问题、样例、快捷任务？
- [ ] 是否支持文本、语音、图片、文件等多模态输入？
- [ ] 是否能有效收集时间、地点、预算、偏好、限制条件等关键槽位？
- [ ] 当信息不足时，AI 是否会主动澄清，而不是自信乱猜？

### 好的输入体验应该做到

- 用户不用懂 prompt
- 输入越少也能开始
- 输入不完整时能被 AI 引导补齐

---

## 5. 状态与记忆

### Checklist

- [ ] 多轮上下文是否连续？
- [ ] 系统能否正确处理“刚才那个”“第一个方案”“换一个更便宜的”这类指代？
- [ ] 是否支持短期记忆？
- [ ] 是否支持长期偏好沉淀？
- [ ] 用户是否可以管理、关闭或清空记忆？
- [ ] 修改一个条件后，系统能否局部调整，而不是整份答案推倒重来？

### 为什么重要

用户觉得 AI “像不像助手”，很多时候不取决于单次回答质量，而取决于它是否真的记得前文、偏好和约束。

---

## 6. 智能架构

### Checklist

- [ ] 单 Agent 是否已经足够？
- [ ] 是否应该采用“主 Agent + 垂直子 Agent + 工具引擎”的结构？
- [ ] 意图识别是否足够稳定？
- [ ] 是否能按任务类型路由给对应能力模块？
- [ ] 是否有清晰的工具边界和异常处理机制？

### 典型子能力示例

- 酒店 Agent
- 行程 Agent
- 攻略 Agent
- 翻译 Agent
- 订单 Agent
- 售后 Agent

好的架构不是让一个模型什么都做，而是让它先判断，再路由，再统一交付。

---

## 7. 输出设计

### Checklist

- [ ] 输出结果是否不只是大段文字？
- [ ] 是否有卡片、列表、富文本、图文、清单、时间线等结构化视图？
- [ ] 输出是否可编辑？
- [ ] 输出是否支持收藏、复制、分享、跳转执行？
- [ ] 输出是否解释了“为什么这样推荐”？

### 核心原则

AI 产品的结果最好是：

> 自然语言解释 + 结构化结果 + 下一步操作入口

而不是只给一大段看似完整的答案。

---

## 8. 修改与迭代体验

### Checklist

- [ ] 用户拿到答案后，是否可以低成本继续修改？
- [ ] 是否支持“换一个”“更便宜一些”“只保留亲子友好”的局部调整？
- [ ] 是否支持重试、再生成、方案比较？
- [ ] 是否能避免让用户每次都重写全部背景？

### 设计建议

优秀 AI 产品更像“共创过程”，而不是“一次性吐答案”。

---

## 9. 执行闭环

### Checklist

- [ ] AI 是停在建议层，还是能推动到执行层？
- [ ] 能否触发下单、预订、提交、跳转、通知、记录等操作？
- [ ] 工具调用失败时是否可回退？
- [ ] 中断后是否支持续接？

### 判断产品成熟度的关键问题

用户说完一句话后，事情到底推进了多少？

这比“模型回答有多像人”更重要。

---

## 10. 转人工与熔断

### Checklist

- [ ] 哪些场景必须转人工？
- [ ] 高风险/高金额/复杂售后是否有明确的熔断策略？
- [ ] 转人工是否只是“给路径说明”，还是能直接给入口？
- [ ] 上下文是否能透传给人工，避免用户重复描述？

### 原则

成熟产品不是硬撑到底，而是知道什么时候该让人接管。

---

## 11. 数据时效与可信度

### Checklist

- [ ] 数据是否有时效校验？
- [ ] 是否能识别停业、过期、冲突信息？
- [ ] 是否展示来源或时间戳？
- [ ] 不确定时，是否会明确说“不确定”？
- [ ] 推荐是否有理由、风险和替代方案？

### 典型风险

在旅行、餐饮、本地服务、交易等场景中，数据过期比回答笨更致命。

---

## 12. 个性化与隐私

### Checklist

- [ ] 个性化推荐是否真的能被用户感知？
- [ ] 是否有明确的 AI 使用说明？
- [ ] 是否支持关闭个性化推荐？
- [ ] 是否支持查看、管理、清空历史？
- [ ] 是否清晰说明 AI 会记住什么、不会记住什么？

---

## 13. 反馈机制

### Checklist

- [ ] 用户是否能点赞/点踩？
- [ ] 是否有结构化反馈标签？
- [ ] 是否支持文本补充？
- [ ] 反馈是否真的会进入后续优化闭环？

### 建议

反馈系统不要只是摆设。它应该直接服务于：

- Badcase 收集
- Prompt 迭代
- 路由优化
- 工具修正
- UI 体验修补

---

## 14. 评估标准

### Checklist

- [ ] 是否有明确 benchmark？
- [ ] 是否覆盖多约束处理能力？
- [ ] 是否覆盖动态调整能力？
- [ ] 是否覆盖指代消解能力？
- [ ] 是否覆盖工具调用成功率？
- [ ] 是否覆盖任务完成率和业务指标？
- [ ] 是否有长期维护的 badcase 库？

### 不要只看模型“聪不聪明”

真正应该看的，是：

- 是否能识别任务
- 是否能问清问题
- 是否能连续记忆
- 是否能调用正确工具
- 是否能把结果交付到用户手里

---

## 四、最值得借鉴的 5 条 AI 产品设计原则

### 原则 1：先定业务目标，再定 AI 形态

不是先做一个聊天框，再找业务去塞。

### 原则 2：高频刚需能力要前置，而不是埋在对话里

快捷卡片、快捷入口、模板任务，是降低使用门槛的关键。

### 原则 3：结果必须尽量可编辑，而不是只可阅读

生成不是终点，修改成本才决定复用率。

### 原则 4：AI 的专业感，来自连续状态与约束处理能力

能不能理解多条件、能不能承接变化、能不能处理指代，比会不会写长答案更重要。

### 原则 5：好的 AI 产品，最终都要走向执行闭环

如果只能说，不能做，它就更像升级版搜索，而不是真正的智能助手。

---

## 五、一句话总结

> 好的 AI 产品，不是“会聊天”，而是“能持续理解用户约束，并把任务一步步推进到结果”。

如果一个产品同时具备以下五点，通常就值得认真看：

- 定位清楚
- 场景够具体
- 状态连续
- 结果可编辑
- 能推进到执行
